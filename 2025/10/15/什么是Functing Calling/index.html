什么是Function Calling

Function Calling是一种让大语言模型(LLM)能够使用外部工具技术方案。如果一个大模型不会使用工具，那么它就只能作为一个简单的对话机器人，并且只能基于训练时的数据进行回答。

如果你希望大模型能够联网查询实时信息、操作本地文件、调用外部API服务等，就需要赋予它使用工具的能力。Function Calling、MCP(Model Control Program)和A2A(Agent-to-Agent)等都是帮助大模型更好地使用工具技术方案。

Function Calling的工具定义结构

在上面的示例中，TOOLS变量定义了可用的工具，每个工具是一个包含两个主要字段的JSON对象：

  • type：字符串类型，指定工具类型，目前主要使用”function”类型
  • function：对象类型，详细描述了函数的使用方法

对于每个function对象，它包含三个关键字段：

  • name：字符串类型，表示函数名称
  • description：字符串类型，描述函数的用途和功能
  • parameters：JSON Schema格式，用于定义函数接受的参数结构，包括参数类型、描述、是否必需等

Function Calling的工作流程

Function Calling的完整工作流程通常包括以下几个步骤：

  1. 工具定义：开发者定义可供模型使用的工具（如上面的温度查询函数）
  2. 请求发送：将用户问题、上下文和工具定义一起发送给LLM
  3. 决策与调用：LLM分析用户问题，决定是否调用工具，并生成适当的调用参数
  4. 执行工具：系统根据LLM的指示调用相应的工具函数
  5. 结果处理：将工具执行结果返回给LLM
  6. 生成回答：LLM基于工具执行结果，用自然语言向用户提供最终回答

实际应用示例

以下是一个简化的对话流程示例：

  1. 用户提问：”旧金山现在的温度是多少？明天呢？”
  2. LLM分析问题后，识别出需要调用两个工具：
    • 首先调用get_current_temperature获取当前温度
    • 然后调用get_temperature_date获取明天的温度预报
  3. 系统执行这两个工具调用，获取结果
  4. LLM将结果用自然语言总结：”旧金山现在的温度是26.1°C，明天预计为25.9°C”

Function Calling的优势

  1. 扩展模型能力：突破大模型知识库的限制，获取实时、个性化信息
  2. 结构化输出：生成规范的函数调用格式，便于与其他系统集成
  3. 准确性提升：通过调用外部工具获取准确信息，减少模型幻觉
  4. 自动化流程：能够根据用户需求自动执行一系列工具调用

与其他技术方案的对比

与Function Calling类似的技术方案还包括：

  • **MCP (Model Control Program)**：更偏向于模型对工具的编排和控制，注重多步骤任务的规划
  • **A2A (Agent-to-Agent)**：强调多个智能体之间的协作，共同完成复杂任务

相比之下，Function Calling更加轻量和直接，主要关注模型与单个工具的交互过程。

总结

Function Calling是连接大语言模型与外部世界的重要桥梁，通过结构化的工具定义，使模型能够根据用户需求自动选择和调用适当的工具，从而大幅扩展了模型的应用场景和实用价值。无论是构建智能助手、自动化工作流还是复杂的AI应用，Function Calling都是一项关键技术。
