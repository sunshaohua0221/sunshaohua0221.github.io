IntelliJ AI插件CC GUI使用说明

CC GUI 是一个运行在 JetBrains 系 IDE 中的 AI 编程插件，原名 Claude Code GUI。它把 Claude Code 和 OpenAI Codex 放进了 IDEA 的侧边栏里，支持 @文件 上下文、图片输入、Diff 对比、历史会话、Agent 和 MCP 扩展，比较适合长期在 IntelliJ IDEA 里写代码、不想频繁切到终端的开发者。

项目地址：

  • 插件市场：https://plugins.jetbrains.com/plugin/29342-cc-gui-claude-or-codex-
  • GitHub：https://github.com/zhukunpenglinyutong/jetbrains-cc-gui

1. 安装插件

打开 IntelliJ IDEA，进入 Settings -> Plugins，搜索 CC GUI，安装后在 IDE 右侧工具栏里打开 CC GUI 面板即可。

如果是第一次使用，插件通常会提示安装或初始化对应的运行时组件，按界面提示完成即可。建议先确认你本机对应的 CLI 或认证已经可用，再回到 IDEA 内使用。

2. 首次配置

CC GUI 的核心思路不是自己重新造一套账号体系，而是尽量复用你本机已经配置好的 AI 环境。

2.1 使用 Claude Code

2.2 使用 OpenAI Codex

如果你平时已经在本机使用过 Codex，也建议先在终端把登录和配置处理好，再让 CC GUI 复用本地环境。这样更稳定，也更方便排查问题。

2.3 代理与网络

如果你在公司网络、代理网络或者自定义网关环境下使用，优先保证本地 CLI 先跑通。CC GUI 更像是把你已有的能力搬进 IDEA，而不是完全脱离本地环境单独运行。

3. 面板怎么用

安装并配置好之后，CC GUI 的日常使用主要围绕下面几个区域：

  • 会话区：用于查看当前对话、历史会话、收藏会话
  • 输入区：输入需求，也可以附带图片
  • 上下文区：通过 @文件、选中代码、当前编辑器内容给 AI 补充上下文
  • 设置区：切换供应商、权限、安全、外观、统计等配置

如果你之前是用终端版 Claude Code，这里最大的区别就是：代码、对话、Diff、文件跳转都在一个 IDE 里完成，上下文切换会少很多。

5. 进阶能力

5.1 MCP

CC GUI 支持 MCP 服务扩展。简单理解，就是让 AI 不只会“看代码和说话”，还可以在授权范围内调用外部工具。

适合接入的 MCP 一般包括：

  • 浏览器调试
  • Git 操作辅助
  • 文档检索
  • Issue 或知识库系统

如果你希望它帮你联动浏览器、分析页面、辅助联调，MCP 会非常有用。

5.2 Skills 和 Agent

如果你需要更固定的工作流，例如：

  • 新项目初始化规范
  • 提交前代码检查
  • 统一 review 提示词
  • 生成 commit message

那就可以把这些动作沉淀到 Skills 或 Agent 命令里。这样你后面每次都不需要从零描述一遍。

5.3 历史会话和收藏

真实开发里，一个需求往往不是一轮对话就结束。CC GUI 支持历史会话、搜索、收藏和导出，这一点很适合以下场景：

  • 第二天继续昨天的改造任务
  • 对比两个不同实现方案
  • 保留关键排障过程

6. 推荐使用方式

想把 CC GUI 用顺，建议遵循下面几个原则：

  • 大需求先让它分析，再让它执行
  • 复杂改动一定配合 @文件 使用
  • 改代码后一定看 Diff
  • 涉及命令执行、文件修改、外部工具调用时，优先保留权限确认
  • 每个会话只处理一个主题，避免上下文污染

7. 常见问题

7.1 面板能打开，但是发不出消息

先不要怀疑插件，先回到终端检查本地 Claude Code 或 Codex 是否真的能正常使用。大多数问题本质上都是本地认证、模型配置或网络没打通。

7.2 模型报错、400 或找不到模型

优先检查你本地配置文件里的模型名是否正确，尤其是在第三方代理或兼容接口场景下，模型别名和真实模型 ID 不一致时最容易出问题。

7.3 读取不到文件上下文

确认文件已经在工程里打开，或者在对话里明确使用 @文件路径。如果文件名里有空格、目录层级较深，最好直接从插件提供的文件引用能力里选，不要手输。

7.4 AI 改得太多，不敢接

不要让它一次做太大。可以拆成下面这种节奏：

  1. 先分析
  2. 再只改一个模块
  3. 看 Diff
  4. 确认后再继续下一步

这比让它一次性改十几个文件可控得多。

8. 一个适合日常开发的结论

如果你已经习惯 IntelliJ IDEA，又希望把 Claude Code 或 Codex 真正融入编码流程，而不是把 AI 当成一个独立聊天窗口，那么 CC GUI 确实值得一装。

它最有价值的地方，不是“多了一个聊天面板”，而是把 上下文引用、Diff 审核、会话管理、Agent 和 MCP 都放进了 IDE 里。只要你本地的 CLI 和认证环境已经配好，它会比纯终端工作流更顺手，也比网页聊天更接近真实开发现场。
