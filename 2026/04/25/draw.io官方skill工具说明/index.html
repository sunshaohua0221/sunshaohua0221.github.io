2. 安装完成后还要做的事

这一步很关键，很多人装完 skill 之后发现生成流程不顺，问题并不在 skill 本身，而在本地没有把 draw.io Desktop 配好。

安装完成后，建议顺手做下面几件事：

2.1 确认 draw.io Desktop 已安装

如果还没安装，先去官网下载安装：

https://www.drawio.com/

官方 skill 在导出 PNG、SVG、PDF 时，会依赖 draw.io Desktop 自带的 CLI。没有桌面版，通常就只能生成 .drawio 文件，不能直接导出图片。

3. 怎么用

官方 drawio skill 的基本思路很简单：

  1. 你提出一个画图需求
  2. AI 生成 draw.io 原生 XML
  3. 写入 .drawio 文件
  4. 如果你明确要求导出格式，再调用 draw.io CLI 输出图片或 PDF

3.3 导出 SVG 或 PDF

如果导出成功，最终拿到的通常是类似下面这种文件名：

  • order-service.drawio.png
  • user-center.drawio.svg
  • payment-process.drawio.pdf

这种双后缀命名的好处是很明确：它本质上还是 draw.io 图导出的产物，而且通常带着内嵌 XML，可再次编辑。

4. 适合画什么图

官方 skill 比较适合下面这些常见开发文档场景：

  • 系统架构图
  • 微服务关系图
  • 流程图
  • ER 图
  • 时序图
  • 类图
  • 网络拓扑图
  • 页面草图和线框图

如果你本来就经常在文档里补这些图，那么这个 skill 的价值会很明显，因为它能把“描述需求 -> 生成初稿 -> 人工微调”这条链路缩短很多。

5. 一个更实用的使用建议

不要一上来就说“帮我画个架构图”，最好把下面这些信息说清楚：

  • 画的是什么系统
  • 主要模块有哪些
  • 模块之间怎么连
  • 你要流程图、架构图还是 ER 图
  • 最终要不要导出成图片

6. 总结

如果你的目标是让 AI 帮你生成真正可编辑的 draw.io 图，而不是临时演示图，那么官方 jgraph/drawio-mcp 的 skill-cli 方案基本就是当前最稳的一种选择。

安装方法也不复杂，本质上就是两步：

  1. 从官方仓库拿到 skill-cli/drawio/SKILL.md
  2. 复制到 ~/.claude/skills/drawio/SKILL.md

然后再把 draw.io Desktop 配好，尤其是：

  • 安装桌面版
  • 打开 Light 主题
  • 打开 Autosave

这三步做完之后，整体体验会顺很多。后面无论是画架构图、流程图，还是做系统设计文档，都会比手工从零拖图省事不少。
