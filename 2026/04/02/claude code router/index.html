claude code router

1. 介绍

claude-code-router 的思路是：

  • 继续使用官方 Claude Code（CLI能力、工作流、工具链都不变）
  • 在模型层通过路由转发，把请求桥接到你自定义的模型服务（例如 Codex/OpenAI 兼容接口）

也就是你说的“官方能力不变，模型路由劫持到自定义模型”。

6. 字段解释

  • PORT: 本地 router 监听端口（示例是 3456）
  • NON_INTERACTIVE_MODE: 非交互模式，便于稳定转发
  • OPENAI_API_KEY: OpenAI/Codex key（按你的网关要求填写）
  • transformers: 请求/响应转换插件
  • Providers: 上游模型提供方列表
    • name: 提供方名字
    • api_base_url: 上游接口地址
    • api_key: 上游 key
    • models: 可用模型
  • Router.default: 默认路由到哪个提供方+模型

7. 使用流程（实践）

  1. 安装 claude-code-router
  2. 写好 ~/.claude-code-router/config.json
  3. 启动并接管：ccr code
  4. 若配置改动后需要重启：ccr restart
  5. 在 Claude Code 中正常使用；模型实际会被转发到你在 Providers 里定义的 Codex 模型

8. 常见问题

Q1：为什么配置报错？

最常见原因是 config.json 里保留了注释。请确认所有 // 注释已删除。

Q2：为什么连不上上游模型？

重点检查：

  • api_base_url 是否可访问
  • api_key 是否正确
  • models 名称是否和上游一致
  • 本地 PORT 是否被占用

Q3：这个方案是否“违规替换 Claude Code”？

不是。Claude Code 客户端本身仍是官方的；只是通过路由层把模型请求转发到你自定义后端。
