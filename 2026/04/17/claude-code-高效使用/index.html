Claude Code 高效使用：命令、会话与工具组合

要把 Claude Code 用得顺手，往往不取决于「只会敲一个 claude」，而是把启动方式、会话内命令、输入技巧和周边工具组合起来。下面按场景整理，便于查阅与落地。

核心结论

高效使用依赖特定命令与工具组合，而不是单一基础命令；权限模式与上下文管理对长任务尤其重要。

启动与权限模式

基础启动

在终端执行 claude 并回车即可进入交互会话（可按需在项目目录下启动，以便模型理解仓库上下文）。

进阶：跳过权限确认（高风险，仅限隔离环境）

进阶：自动模式（适合长任务）

自动模式会在执行前做安全判断，尽量自动放行「相对安全」的操作，减少反复按确认键，适合耗时较长的重构或批量任务。启动示例：

会话内命令与上下文

能力 说明
/model 切换模型；中途切换可能使 KV 缓存失效，长对话下会多耗 token、略损连贯性，重要任务前尽量先定好模型。
effort 用于设置「思考强度」类选项（具体取值以当前 CLI 提示为准），在难推理任务上可酌情调高。
/clear 彻底清空当前会话上下文，从零开始对话。
/compact 压缩上下文，把历史折叠为摘要，缓解窗口占满问题。
ESC 打断当前进行中的操作；连按两下在部分交互里可起到回退/取消一类效果（以终端实际行为为准）。

特殊输入技巧

「By the way」类插入

在长任务执行过程中，若需要临时插入与主线无关的小问题，可使用约定式的 by the way 说法（或产品内等价的「旁路提问」能力），让补充问题尽量不污染主任务上下文。具体语法以当前版本说明为准。

感叹号执行终端命令

在支持的输入方式下，行首使用 ! 可直接执行终端命令（例如 !pwd 打印当前目录），减少在 Claude 与 shell 之间来回切换。

工具与插件

Claude HUD（上下文与 Token 可视化）

Claude HUD 是面向 Claude Code 的插件，可在终端中展示上下文占用、剩余 token 等信息，便于在长会话里判断何时该 /compact 或拆会话。安装与启用方式见仓库 README（常见流程包含在插件市场中添加并执行 setup）。

输入法与系统层体验

  • 豆包输入法等支持语音转文字的工具，可减轻长提示的手工输入成本（注意敏感环境不宜使用联网语音）。
  • 灵动岛（或系统级 Agent 状态展示）：在支持的设备或配套软件上，可切换/查看 Agent 运行状态，与终端里的会话形成互补（具体能力取决于系统与第三方实现）。

小结

  • 隔离环境再考虑 --dangerously-skip-permissions；日常长任务可优先了解 --permission-mode auto 是否适用。
  • 模型与 /compact、/clear 的配合比「一味加对话」更能控成本。
  • HUD + 语音输入 + 系统状态展示属于「效率叠 buff」，按自己的安全与习惯选用即可。

以上内容随 Claude Code 版本迭代可能变化，以官方文档与当前 CLI --help 为准。
